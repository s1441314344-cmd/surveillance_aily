# MES系统工作流编排功能重设计需求文档

## 1. 项目背景

### 1.1 现状分析
当前MES系统针对饮料生产的工作流编排功能基于Camunda BPMN实现，存在以下问题：
- **操作体验差**：界面交互复杂，用户学习成本高
- **视觉设计落后**：界面美观度不足，影响用户体验
- **流程编排繁琐**：添加、编辑、调整节点步骤复杂

### 1.2 目标
参考飞书多维表的工作流交互方式，重新设计一套直观、高效、美观的工作流编排系统。

---

## 2. 需求概述

### 2.1 核心目标
- 简化工作流编排操作，降低用户学习成本
- 提升界面美观度和用户体验
- 支持并行流程编排
- 提供便捷的节点内容编辑能力

### 2.2 目标用户
- 工艺工程师：负责设计生产工艺流程
- 生产管理人员：查看和管理工作流
- 系统管理员：维护和优化工作流模板

## 3. 功能需求

### 3.1 整体布局设计

#### 3.1.1 三栏式布局
```
┌─────────────────────────────────────────────────────────────────┐
│  顶部导航栏                                                        │
├──────────────┬──────────────────────────────┬───────────────────┤
│              │                              │                   │
│  左侧节点库   │      中央画布区域             │   右侧属性面板     │
│  (可收起)     │      (可视化流程编排)          │   (可收起)        │
│              │                              │                   │
│  - 业务节点   │                              │   - 节点属性       │
│  - 系统节点   │                              │   - 参数配置       │
│  - 并行节点   │                              │   - 条件设置       │
│              │                              │                   │
└──────────────┴──────────────────────────────┴───────────────────┘
```

#### 3.1.2 布局交互规则
- 左侧节点库默认展开，支持点击收起/展开
- 右侧属性面板默认收起，选中节点时自动展开
- 中央画布支持缩放（50%-200%）和平移
- 支持全屏模式编辑

### 3.2 左侧节点库

#### 3.2.1 节点分类
| 分类 | 节点类型 | 图标 | 说明 |
|------|---------|------|------|
| **业务节点** | 确认投料口 | 📦 | 确认原料投入位置 |
| | 加水 | 💧 | 添加指定量的水 |
| | 搅拌 | 🔄 | 搅拌混合原料 |
| | 投料 | ⬇️ | 投入原料 |
| | 抽取 | ⬆️ | 抽取物料 |
| | 赶水 | 💨 | 驱赶水分 |
| | 预混 | 🥣 | 预混合处理 |
| | 复水 | ♻️ | 复水处理 |
| | 均质 | ⚖️ | 均质化处理 |
| **系统节点** | 开始 | ● | 流程起始节点 |
| | 结束 | ◉ | 流程结束节点 |
| | 并行开始 | ⫘ | 并行分支起点 |
| | 并行结束 | ⫘ | 并行分支汇合点 |
| | 等待 | ⏸️ | 等待条件满足 |

#### 3.2.2 节点库交互
- **拖拽添加**：从节点库拖拽节点到画布
- **点击添加**：点击节点后，在画布点击位置添加
- **搜索筛选**：支持按名称搜索节点
- **最近使用**：显示最近使用的节点（Top 5）
- **收藏节点**：支持收藏常用节点

### 3.3 中央画布区域

#### 3.3.1 流程展示方式
采用**垂直时间线+卡片式**布局，替代传统BPMN图形：

```
    ┌─────────┐
    │  开始   │
    └────┬────┘
         │
    ┌────▼────┐
    │   加水  │
    └────┬────┘
         │
    ┌────▼────┐
    │   搅拌  │
    └────┬────┘
         │
    ┌────▼────┐
    │ 并行开始 │
    └────┬────┘
    ┌────┴────┐
    │         │
┌───▼───┐ ┌───▼───┐
│  预混  │ │  复水  │
└───┬───┘ └───┬───┘
    │         │
    └────┬────┘
    ┌────▼────┐
    │ 并行结束 │
    └────┬────┘
         │
    ┌────▼────┐
    │   结束  │
    └─────────┘
```

#### 3.3.2 节点卡片设计
每个节点以卡片形式展示：

```
┌─────────────────────────────────────┐
│  💧  加水                    [⋯]   │  ← 图标+名称+操作菜单
├─────────────────────────────────────┤
│  工序: 加水                         │  ← 关键信息摘要
│  岗位: 中控技术员                    │
│  参数: 2个                          │
└─────────────────────────────────────┘
```

**卡片状态样式：**
- 默认状态：白色背景，灰色边框
- 选中状态：蓝色边框，轻微阴影
- 悬停状态：显示操作按钮（编辑、删除、添加连线）
- 错误状态：红色边框，显示错误提示

#### 3.3.3 连线设计
- **直线连接**：默认使用直线连接节点
- **自动路由**：连线自动避让节点，避免交叉
- **连线样式**：
  - 正常流程：实线箭头
  - 并行分支：虚线（可选不同颜色）
  - 条件分支：带标签的连线

#### 3.3.4 画布交互
| 操作 | 功能 |
|------|------|
| 单击节点 | 选中节点，展开右侧属性面板 |
| 双击节点 | 进入节点编辑模式 |
| 拖拽节点 | 调整节点位置 |
| 拖拽连线端点 | 调整连线连接关系 |
| 滚轮 | 画布缩放 |
| 空格+拖拽 | 画布平移 |
| 右键菜单 | 删除节点、复制节点、添加注释 |
| Delete键 | 删除选中节点 |
| Ctrl+C/V | 复制/粘贴节点 |

### 3.4 并行流程支持

#### 3.4.1 并行分支展示
```
         │
    ┌────▼────┐
    │ 并行开始 │  ← 菱形节点，带"+"号
    └────┬────┘
    ┌────┴────┐
    │         │
分支1│         │分支2
    │         │
┌───▼───┐ ┌───▼───┐
│ 节点A  │ │ 节点B  │
└───┬───┘ └───┬───┘
    │         │
    └────┬────┘
    ┌────▼────┐
    │ 并行结束 │  ← 菱形节点，带"-"号
    └────┬────┘
         │
```

#### 3.4.2 并行分支操作
- **添加分支**：在并行开始节点上点击"添加分支"
- **删除分支**：删除分支上所有节点后自动移除分支
- **分支排序**：拖拽调整分支顺序
- **同步设置**：设置分支是否需要全部完成或任意完成

#### 3.5.2 参数配置功能
- **新增参数**：点击"+新增参数"添加
- **参数字段**：序号、参数名称、单位、默认值、上下限、是否必填
- **参数编辑**：行内编辑，实时保存
- **参数删除**：单行删除，支持批量删除
- **参数排序**：拖拽调整顺序

#### 3.5.3 执行条件配置
- **条件类型**：
  - 前置节点完成
  - 时间条件
  - 参数条件（如温度达到设定值）
  - 自定义条件
- **条件组合**：支持AND/OR逻辑组合

## 4. 交互流程设计

### 4.1 创建新流程
```
1. 点击"新建流程" → 弹出模板选择对话框
2. 选择"空白流程"或"应用模板"
3. 进入编排界面
4. 从左侧拖拽节点到画布
5. 系统自动连线（或手动连线）
6. 点击节点编辑属性
7. 保存并发布
```

### 4.2 编辑现有流程
```
1. 打开流程列表
2. 点击流程进入编辑模式
3. 选中节点进行编辑/删除
4. 拖拽添加新节点
5. 调整节点位置和连线
6. 保存修改
```

### 4.3 添加并行分支
```
1. 选中需要分叉的节点
2. 点击"添加并行分支"或拖拽"并行开始"节点
3. 系统自动创建并行开始和并行结束节点
4. 在各分支上添加节点
5. 配置分支完成条件
```

## 6. 响应式设计

### 6.1 适配方案
- **桌面端（≥1440px）**：三栏布局，完整功能
- **笔记本（1024-1439px）**：三栏布局，可收起侧边栏
- **平板（768-1023px）**：双栏布局，节点库可收起
- **不支持移动端**：工作流编排功能仅限桌面端

## 7. 性能要求

### 7.1 加载性能
- 首屏加载时间 ≤ 2秒
- 节点拖拽响应时间 ≤ 100ms
- 属性面板展开动画 ≤ 200ms

### 7.2 渲染性能
- 支持同时渲染100+节点不卡顿
- 画布缩放/平移流畅（60fps）

## 8. 数据接口需求

### 8.1 节点数据模型
```typescript
interface WorkflowNode {
  id: string;              // 节点唯一标识
  type: NodeType;          // 节点类型
  name: string;            // 节点名称
  icon: string;            // 节点图标
  position: { x: number; y: number };  // 画布位置
  properties: {
    processName: string;   // 工序名称
    position: string;      // 岗位
    taskType: string;      // 任务类型
    description: string;   // 工序描述
    parameters: Parameter[];  // 参数列表
    conditions: Condition[];  // 执行条件
  };
  nextNodes: string[];     // 下游节点ID列表
  prevNodes: string[];     // 上游节点ID列表
}
```

### 8.2 流程数据模型
```typescript
interface Workflow {
  id: string;
  name: string;
  version: string;
  status: 'draft' | 'published';
  nodes: WorkflowNode[];
  createTime: Date;
  updateTime: Date;
  creator: string;
}
```
