## 1. 项目概述

### 1.1 项目背景

当前基于飞书Aily平台的智能巡检系统存在以下问题：

- **依赖性强**：系统依赖Aily平台和飞书多维表格，架构复杂
- **调试困难**：跨平台调用导致问题排查耗时
- **数据分散**：检测数据存储在飞书多维表格，访问不便
- **第三方限制**：受限于Aily平台的功能和配额
- **部署复杂**：需要同时维护Aily配置和飞书API

### 1.2 项目目标

设计并实现一套**完全独立运行的智能巡检系统**，实现：

- 完全脱离Aily平台，不依赖任何第三方AI服务（仅使用智谱GLM-4V）
- 完全脱离飞书多维表格，使用本地SQLite数据库存储所有数据
- 提供友好的Web界面，支持摄像头配置、规则管理、图片检测、记录查询
- 支持真实的大模型图像分析调用（智谱GLM-4V）
- 支持真实的数据持久化存储
- 支持真实的前端HTTP请求调用

### 1.3 项目范围

**包含范围：**
- 摄像头配置管理（RTSP地址、抽帧参数）
- 巡检规则配置（提示词模板CRUD）
- 本地图片上传检测
- 智谱GLM-4V大模型图像分析
- SQLite数据库持久化存储
- 检测记录查询和导出

**不包含范围：**
- 飞书集成功能（AilyService、FeishuService）
- 实时视频流处理（RTSP抽帧定时任务）
- 微信/其他第三方集成
- 移动端原生应用

## 2. 用户角色分析

### 2.1 用户角色定义

| 角色 | 描述 | 权限范围 | 典型操作 |
|------|------|----------|----------|
| **系统管理员** | 负责系统配置和维护 | 全部功能 | 摄像头配置、规则管理、系统设置 |
| **巡检操作员** | 执行日常巡检任务 | 检测和查询 | 图片上传检测、查看检测记录 |

### 2.2 用户场景

#### 场景1：系统管理员配置新摄像头

```
角色：系统管理员
场景：配置新的监控摄像头
触发条件：新增监控点位
输入：
  - 摄像头编码（唯一）
  - 摄像头名称

  - 位置描述
  - 抽帧频率（秒）
预期结果：
  - 摄像头信息保存到数据库

  - 前端显示更新后的列表

```
角色：系统管理员
场景：创建新的巡检规则
触发条件：需要新增检测类型
输入：
  - 规则编码（唯一）
  - 规则名称
  - 提示词内容（完整prompt）
  - 规则描述
预期结果：
  - 规则保存到数据库
  - 返回规则ID
  - 规则可在检测时选择

角色：巡检操作员

  - 图片文件（支持JPG/PNG/BMP）

  - 图片上传到服务器
  - 调用智谱GLM-4V进行分析

  - 检测记录保存到数据库

### 3.2 功能需求详细定义

#### FR-001: 摄像头配置管理

| 属性 | 值 |
|------|-----|
| **需求ID** | FR-001 |
| **需求名称** | 摄像头配置管理 |
| **需求类型** | 功能需求 |
| **优先级** | Must Have |
| **描述** | 系统 SHALL 提供摄像头的完整配置管理功能 |

**详细功能列表：**

1. **FR-001.1**: 添加摄像头
   - 输入：编码、名称、RTSP地址、位置、频率
   - 处理：数据验证、唯一性检查、数据库保存
   - 输出：成功返回摄像头ID，失败返回错误信息

2. **FR-001.2**: 编辑摄像头
   - 输入：摄像头ID、待更新字段
   - 处理：数据验证、数据库更新
   - 输出：成功返回更新后的摄像头信息

3. **FR-001.3**: 删除摄像头
   - 输入：摄像头ID
   - 处理：检查关联记录、数据库删除
   - 输出：成功返回删除状态

4. **FR-001.4**: 查询摄像头列表
   - 输入：分页参数（page, page_size）
   - 处理：数据库查询
   - 输出：摄像头列表和总数

5. **FR-001.5**: 获取摄像头详情
   - 输入：摄像头ID
   - 处理：数据库查询
   - 输出：摄像头详细信息

6. **FR-001.6**: 测试RTSP连接
   - 输入：RTSP地址
   - 处理：尝试建立连接（超时10秒）
   - 输出：连接成功/失败状态

**业务规则：**
- 摄像头编码必须唯一
- RTSP地址格式：rtsp://[user:pass@]host[:port]/path
- 抽帧频率范围：10秒 - 3600秒
- 删除摄像头前需检查是否有检测记录关联

**依赖关系：**
- 依赖数据库模块（db.py）

### 5.2 数据表详细定义

#### cameras表

```sql
CREATE TABLE cameras (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code TEXT UNIQUE NOT NULL,
    name TEXT NOT NULL,
    rtsp_url TEXT,
    location TEXT,
    frequency INTEGER DEFAULT 60,
    status TEXT DEFAULT 'active' CHECK(status IN ('active', 'inactive')),
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 自增主键 |
| code | TEXT | UNIQUE, NOT NULL | 摄像头编码 |
| name | TEXT | NOT NULL | 摄像头名称 |
| rtsp_url | TEXT | NULL | RTSP地址 |
| location | TEXT | NULL | 位置描述 |
| frequency | INTEGER | DEFAULT 60 | 抽帧频率（秒） |
| status | TEXT | DEFAULT 'active' | 状态 |
| created_at | TEXT | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TEXT | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

#### rules表

```sql
CREATE TABLE rules (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code TEXT UNIQUE NOT NULL,
    name TEXT NOT NULL,
    prompt_content TEXT NOT NULL,
    description TEXT,
    status TEXT DEFAULT 'active' CHECK(status IN ('active', 'inactive')),
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 自增主键 |
| code | TEXT | UNIQUE, NOT NULL | 规则编码 |
| name | TEXT | NOT NULL | 规则名称 |
| prompt_content | TEXT | NOT NULL | 提示词内容 |
| description | TEXT | NULL | 规则描述 |
| status | TEXT | DEFAULT 'active' | 状态 |
| created_at | TEXT | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TEXT | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

#### detection_records表

```sql
CREATE TABLE detection_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    camera_id INTEGER,
    rule_id INTEGER NOT NULL,
    image_path TEXT,
    result_image_path TEXT,
    llm_result TEXT,
    has_violation INTEGER DEFAULT 0,
    detect_time TEXT DEFAULT CURRENT_TIMESTAMP,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (camera_id) REFERENCES cameras(id) ON DELETE SET NULL,
    FOREIGN KEY (rule_id) REFERENCES rules(id) ON DELETE CASCADE
);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 自增主键 |
| camera_id | INTEGER | FOREIGN KEY | 关联摄像头ID |
| rule_id | INTEGER | FOREIGN KEY, NOT NULL | 关联规则ID |
| image_path | TEXT | NULL | 原始图片路径 |
| result_image_path | TEXT | NULL | 结果图片路径 |
| llm_result | TEXT | NULL | LLM分析结果 |
| has_violation | INTEGER | DEFAULT 0 | 是否违规（0/1） |
| detect_time | TEXT | DEFAULT CURRENT_TIMESTAMP | 检测时间 |
| created_at | TEXT | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

## 10. 潜在需求冲突与模糊点识别

### 10.1 冲突识别

| 冲突ID | 冲突描述 | 影响分析 | 解决方案 |
|--------|----------|----------|----------|
| **CON-001** | YOLO与LLM同时启用可能性能下降 | 当前版本禁用YOLO | 采用分阶段策略：YOLO快速初筛 + LLM详细分析（未来版本） |
| **CON-002** | 大模型响应时间不稳定 | 用户体验 | 添加超时配置、增加loading状态 |
| **CON-003** | SQLite不适合高并发写入 | 未来扩展瓶颈 | 当前版本接受限制，文档说明扩展建议 |

### 10.2 模糊点识别

| 模糊ID | 模糊描述 | 当前理解 | 需确认 |
|--------|----------|----------|--------|
| **AMB-001** | 违规判定标准 | LLM返回结果中包含"违规"字样即标记 | 是否需要标准化JSON格式输出？ |
| **AMB-002** | 摄像头实时检测 | 当前版本仅支持手动上传图片 | RTSP定时抽帧是否需要？ |
| **AMB-003** | 检测结果图片标注 | YOLO禁用，暂不生成标注图 | 未来是否需要YOLO标注？ |
| **AMB-004** | 用户认证授权 | 当前版本无认证 | 是否需要添加用户系统？ |
| **AMB-005** | 规则提示词模板 | 自由文本，无结构化 | 是否需要预设模板库？ |

### 10.3 已知限制

| 限制ID | 限制描述 | 影响 | 缓解措施 |
|--------|----------|------|----------|
| **LIM-001** | SQLite单文件数据库 | 并发写入受限 | 文档说明并发限制 |
| **LIM-002** | 智谱API配额限制 | 每日调用次数限制 | 配置API Key监控 |
| **LIM-003** | 图片存储空间 | 磁盘空间占用 | 定期清理或导出删除 |
| **LIM-004** | 无用户认证 | 安全性较低 | 内部网络使用 |
