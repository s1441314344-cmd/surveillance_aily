# 摄像头设备页

## 1. 需求概要

### 模块定位
摄像头设备页负责维护摄像头基础信息和设备级操作入口，是摄像头中心的设备资产管理页。

### 目标
- 维护设备基础信息
- 提供抓拍和录像入口
- 支撑其他摄像头子页的设备上下文

### 页面范围
- 设备列表与编辑
- 新建设备
- 抓拍与录像入口

### 模块依赖
- **全局规则**：`/prd/需求总览.md` §3 角色与权限、§5.1 共享实体
- **共享上下文**：`/prd/modules/cameras-索引.md` §3

### Demo 实现状态

| 功能点 | Demo 状态 | 数据来源 | 说明 |
|--------|-----------|----------|------|
| 设备列表 | ✅ 已实现 | 真实后端 | 展示设备与状态摘要 |
| 新建/编辑设备 | ✅ 已实现 | 表单 + 后端 | 支持设备基础配置 |
| 抓拍入口 | ✅ 已实现 | 真实后端 | 用于快速验证设备可用性 |
| 录像入口 | ✅ 已实现 | 真实后端 | 用于媒体采集 |

### 一期不做
- 当前不做批量导入设备
- 当前不做跨项目设备共享

## 2. 页面功能总览
- 设备管理：查看、创建、编辑设备
- 设备选择：通过侧边栏切换当前摄像头
- 设备操作：快速抓拍和录像

## 3. 功能模块

### 3.1 设备维护
#### 3.1.1 功能目的
维护摄像头的基础配置，使其可被任务和监测流程使用。

#### 3.1.2 触发条件
用户新建摄像头或编辑当前选中设备。

#### 3.1.3 前置条件
用户具备系统管理员权限。

#### 3.1.4 主流程
1. 用户在侧边栏选择设备或进入新建设备态。
2. 页面展示当前设备表单。
3. 用户修改名称、地址、协议、存储路径等基础信息。
4. 保存成功后刷新设备列表和当前详情。

#### 3.1.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.1.6 状态与反馈
- 默认态：展示当前设备配置
- 成功：提示保存成功
- 失败：提示保存失败原因

#### 3.1.7 交互与展示
- 新建设备和编辑设备使用同一套表单
- 当前设备标题和关键信息应在表单顶部明确展示

#### 3.1.8 异常与边界
- 保存中禁止重复提交
- 选中设备被删除后应退出到空态或新建态

### 3.2 抓拍与录像入口
#### 3.2.1 功能目的
为设备联调和媒体采集提供快速入口。

#### 3.2.2 触发条件
用户点击抓拍或录像相关操作。

#### 3.2.3 前置条件
当前存在已保存摄像头。

#### 3.2.4 主流程
1. 用户在设备页触发抓拍或开始录像。
2. 系统执行设备级媒体操作。
3. 成功后把结果同步到媒体页可见范围。

#### 3.2.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.2.6 状态与反馈
- 加载中：当前操作按钮禁用
- 成功：提示操作成功
- 失败：提示设备操作失败原因

#### 3.2.7 交互与展示
- 抓拍和录像应作为设备级动作显式展示

#### 3.2.8 异常与边界
- 未保存的新建设备不可执行设备级媒体操作

## 4. 页面级共享规则
- 当前页以 `selectedCameraId` 作为设备上下文来源
- 抓拍和录像结果归入媒体页统一查看，不在设备页长期承载历史媒体列表

## 5. 数据模型
### 5.1 依赖共享实体
- `Camera`
- `CameraStatusLog`
- `CameraMedia`

### 5.2 页面私有对象
| 字段 | 含义 |
|------|------|
| `selectedCameraId` | 当前选中设备 |
| `createCameraMode` | 新建设备态标识 |
