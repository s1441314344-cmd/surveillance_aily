# 用户与权限

## 1. 需求概要

### 模块定位
用户与权限页是系统治理入口，用于管理用户状态、角色分配和访问边界。

### 目标
- 管理系统用户
- 管理角色分配和启停状态
- 为其他受控页面提供统一权限来源

### 页面范围
- 用户列表
- 用户统计
- 新建用户
- 用户启停

### 模块依赖
- **全局规则**：`/prd/需求总览.md` §3 角色与权限
- **关联模块**：`/prd/modules/auth-login.md` §5

### Demo 实现状态

| 功能点 | Demo 状态 | 数据来源 | 说明 |
|--------|-----------|----------|------|
| 用户列表 | ✅ 已实现 | 真实后端 | 展示用户名、显示名、角色和状态 |
| 新建用户 | ✅ 已实现 | 真实后端 | 支持角色分配 |
| 用户启停 | ✅ 已实现 | 真实后端 | 支持 active 状态切换 |
| 用户统计 | ✅ 已实现 | 前端聚合 | 展示数量概览 |

### 一期不做
- 当前不做细粒度权限矩阵编辑器

## 2. 页面功能总览
- 用户列表：查看当前系统用户
- 用户统计：快速查看用户规模和状态分布
- 新建用户：创建新账号并分配角色
- 用户启停：控制账号是否可登录

## 3. 功能模块

### 3.1 用户列表与统计
#### 3.1.1 功能目的
让系统管理员了解当前账号体系和角色分布。

#### 3.1.2 触发条件
用户进入用户与权限页。

#### 3.1.3 前置条件
用户具备系统管理员权限。

#### 3.1.4 主流程
1. 页面读取用户列表。
2. 页面展示用户统计概览。
3. 用户选择当前关注用户。

#### 3.1.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.1.6 状态与反馈
- 空态：当前没有用户
- 失败：提示列表加载失败

#### 3.1.7 交互与展示
- 列表需展示角色和启停状态

#### 3.1.8 异常与边界
- 当前用户无权限时不展示管理能力

### 3.2 新建与启停
#### 3.2.1 功能目的
让管理员可以直接维护账号生命周期。

#### 3.2.2 触发条件
用户提交新建表单或执行启停操作。

#### 3.2.3 前置条件
用户具备系统管理员权限。

#### 3.2.4 主流程
1. 用户填写新用户信息和角色。
2. 保存成功后用户列表刷新。
3. 用户可对已有账号执行启停操作。

#### 3.2.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.2.6 状态与反馈
- 成功：提示操作成功
- 失败：提示失败原因

#### 3.2.7 交互与展示
- 新建表单与列表同屏展示

#### 3.2.8 异常与边界
- 保存中禁止重复提交

## 4. 页面级共享规则
- 角色定义与登录页共享同一套业务语义
