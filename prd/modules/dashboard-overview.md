# 总览看板

## 1. 需求概要

### 模块定位
总览看板用于按任务、记录和复核结果统一展示系统运行状态，是主链结果的聚合入口。

### 目标
- 用统一筛选条件观察主链运行状态
- 提供核心指标、趋势和异常案例入口
- 支持按看板定义套用预设筛选条件

### 页面范围
- 总览筛选
- 核心指标卡
- 趋势图
- 异常案例表

### 模块依赖
- **全局规则**：`docs/PRD/需求总览.md` §4 全局规则与共享状态、§5 共享实体与统计口径
- **关联模块**：`docs/PRD/modules/dashboards-config.md` §3.1、`docs/PRD/modules/records-center.md` §3.1

### Demo 实现状态

| 功能点 | Demo 状态 | 数据来源 | 说明 |
|--------|-----------|----------|------|
| 总览筛选 | ✅ 已实现 | `frontend-v2` + `backend-v2` | 支持按看板、策略、模型、异常类型和时间筛选 |
| 核心指标卡 | ✅ 已实现 | 聚合接口 | 展示任务、记录、复核等核心指标 |
| 任务趋势 | ✅ 已实现 | 聚合接口 | 展示时间序列趋势 |
| 异常案例表 | ✅ 已实现 | 聚合接口 | 提供异常案例摘要 |

### 一期不做
- 当前不做拖拽式个性化编排
- 当前不做多看板并排展示

## 2. 页面功能总览
- 筛选条件：统一控制当前看板的数据范围
- 指标总览：展示核心指标和比率
- 趋势分析：按时间维度展示任务运行趋势
- 异常案例：聚合异常案例摘要，支持下钻到记录页面

## 3. 功能模块

### 3.1 筛选条件与看板预设
#### 3.1.1 功能目的
让用户在统一范围内查看总览数据，避免指标口径漂移。

#### 3.1.2 触发条件
用户进入总览页、切换筛选条件或套用看板定义。

#### 3.1.3 前置条件
系统已有可用的看板定义、策略和模型提供方数据。

#### 3.1.4 主流程
1. 页面加载时读取可用看板定义和筛选项。
2. 用户选择看板预设时，系统把预设筛选条件写入当前筛选状态。
3. 用户修改策略、模型、异常类型或时间范围时，系统刷新总览数据。
4. 用户点击重置时恢复默认筛选。

#### 3.1.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.1.6 状态与反馈
- 默认态：展示可选筛选项和当前默认筛选
- 加载中：总览数据与筛选项加载中
- 失败：展示数据加载失败提示

#### 3.1.7 交互与展示
- 筛选区集中放在页面顶部
- 看板定义作为预设入口，不替代细粒度筛选
- 重置操作应显式恢复默认状态

#### 3.1.8 异常与边界
- 筛选项加载失败时，总览区不展示误导性数据
- 加载过程中避免重复触发无意义刷新

### 3.2 核心指标与趋势展示
#### 3.2.1 功能目的
让用户快速判断主链运行质量和波动趋势。

#### 3.2.2 触发条件
筛选条件加载完成或被修改后。

#### 3.2.3 前置条件
存在可用于统计的任务和记录数据。

#### 3.2.4 主流程
1. 系统按当前筛选条件读取汇总指标和趋势点。
2. 页面展示总任务数、总记录数、待复核数、结构化成功率、异常率和复核准确率。
3. 页面按时间维度展示任务趋势。
4. 若当前范围内没有数据，展示空态而不是零值误导。

#### 3.2.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.2.6 状态与反馈
- 默认态：展示上一次成功加载的数据
- 加载中：指标区和趋势区进入骨架屏或加载态
- 空态：明确说明当前筛选范围暂无数据

#### 3.2.7 交互与展示
- 指标卡优先展示绝对值与关键比率
- 趋势图以时间维度连续展示，不混入多个不相关指标

#### 3.2.8 异常与边界
- 无数据时显示空态而不是全部显示 `0`
- 聚合接口失败时显示统一错误提示

### 3.3 异常案例列表
#### 3.3.1 功能目的
让用户在总览页直接看到需要关注的异常案例。

#### 3.3.2 触发条件
总览数据加载完成。

#### 3.3.3 前置条件
当前筛选范围内存在异常或失败记录。

#### 3.3.4 主流程
1. 系统按当前筛选条件读取异常案例列表。
2. 页面展示策略名、异常摘要、结果状态、复核状态和创建时间。
3. 用户据此决定是否进入记录页或反馈页继续处理。

#### 3.3.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.3.6 状态与反馈
- 空态：当前没有异常案例
- 失败：异常案例区独立展示错误状态

#### 3.3.7 交互与展示
- 列表按异常优先级和时间新旧排序
- 摘要需要在列表内保持可扫读性

#### 3.3.8 异常与边界
- 摘要过长时截断展示
- 列表失败不应影响已成功加载的指标区

## 4. 页面级共享规则
- 总览页的筛选状态是本页内部共享状态，不写回全局会话
- 看板预设只影响当前页面筛选，不修改看板定义本身
