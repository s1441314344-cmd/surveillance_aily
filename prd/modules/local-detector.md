# 本地检测

## 1. 需求概要

### 模块定位
本地检测页是 `local-detector` 侧车服务的调试入口，用于查看服务状态、维护配置并执行单图验证。

### 目标
- 检查本地检测服务健康状态
- 维护本地检测配置
- 支持上传图和摄像头拍照调试

### 页面范围
- 服务健康状态
- 服务配置
- 单图调试
- 调试历史

### 模块依赖
- **全局规则**：`docs/PRD/需求总览.md` §4.3 扩展子域定级
- **关联模块**：`docs/PRD/modules/cameras-devices.md` §3.2

### Demo 实现状态

| 功能点 | Demo 状态 | 数据来源 | 说明 |
|--------|-----------|----------|------|
| 健康检查 | ✅ 已实现 | `local-detector` | 周期性读取健康状态 |
| 配置维护 | ✅ 已实现 | `local-detector` | 支持模型配置维护 |
| 上传图调试 | ✅ 已实现 | `local-detector` | 支持上传本地图像调试 |
| 摄像头拍照调试 | ✅ 已实现 | `local-detector` + 主 API | 复用摄像头抓拍能力 |

### 一期不做
- 当前不把本地检测结果直接纳入正式任务归档

## 2. 页面功能总览
- 服务状态：查看健康和配置状态
- 配置维护：更新本地检测配置
- 单图调试：上传文件或从摄像头抓拍进行验证
- 调试记录：查看最近调试结果

## 3. 功能模块

### 3.1 服务状态与配置维护
#### 3.1.1 功能目的
保证本地检测服务可用且配置正确。

#### 3.1.2 触发条件
用户进入页面或保存配置。

#### 3.1.3 前置条件
用户具备系统管理员权限。

#### 3.1.4 主流程
1. 页面读取健康状态和当前配置。
2. 用户修改配置并保存。
3. 系统更新配置并刷新健康状态。

#### 3.1.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.1.6 状态与反馈
- 成功：提示配置已保存
- 失败：按失败类型提示保存失败、服务不可用、连接超时或访问未授权

#### 3.1.7 交互与展示
- 配置项和健康状态需分块展示

#### 3.1.8 异常与边界
- 服务不可用时仍保留只读状态信息
- 服务不可用、连接超时或访问未授权时，需要给出明确提示，不使用模糊的通用失败文案

### 3.2 单图调试
#### 3.2.1 功能目的
快速验证本地检测服务在当前配置下的检测效果。

#### 3.2.2 触发条件
用户上传图片或从摄像头抓拍后发起检测。

#### 3.2.3 前置条件
当前已存在调试输入图片，且本地检测服务健康状态为 `ready=true`。

#### 3.2.4 主流程
1. 用户上传文件或从摄像头抓拍加载图片。
2. 用户设置阈值和规则条件。
3. 系统执行本地检测。
4. 页面展示信号、检测框和调试结果。

#### 3.2.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.2.6 状态与反馈
- 默认态：未加载图片
- 成功：展示检测结果
- 失败：按失败类型提示检测失败、服务不可用、连接超时或访问未授权

#### 3.2.7 交互与展示
- 检测结果需包含图像预览和结构化信号结果

#### 3.2.8 异常与边界
- 未加载输入图像时禁止发起检测
- 服务未就绪时禁止发起检测，并继续展示当前健康状态和错误信息
- 调试中禁止重复提交
- 服务不可用、连接超时或访问未授权时，需要给出明确提示，不使用模糊的通用失败文案

## 4. 页面级共享规则
- 本页是治理/调试页，不把检测结果当作正式任务记录
- 从摄像头抓拍加载图片时只复用设备能力，不共享任务上下文

## 5. 数据模型
### 5.1 依赖共享实体
- 本地检测配置
- 本地检测结果
- `Camera`

### 5.2 页面私有对象
| 字段 | 含义 |
|------|------|
| `selectedFile` | 当前调试图片 |
| `latestResult` | 当前检测结果 |
| `records` | 最近调试历史 |
