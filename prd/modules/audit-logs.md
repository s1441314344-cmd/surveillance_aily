# 操作审计

## 1. 需求概要

### 模块定位
操作审计页用于查看后台操作和异常行为轨迹，是治理与安全留痕页面。

### 目标
- 提供写操作审计留痕
- 支持按条件检索审计记录
- 为问题排查和合规提供依据

### 页面范围
- 审计筛选
- 审计列表

### 模块依赖
- **全局规则**：`docs/PRD/需求总览.md` §3 角色与权限

### Demo 实现状态

| 功能点 | Demo 状态 | 数据来源 | 说明 |
|--------|-----------|----------|------|
| 审计筛选 | ✅ 已实现 | 真实后端 | 支持方法、成功状态、路径、操作人和时间范围 |
| 审计列表 | ✅ 已实现 | 真实后端 | 展示基础审计信息 |

### 一期不做
- 当前不做复杂报表和导出

## 2. 页面功能总览
- 审计筛选：按操作维度缩小范围
- 审计列表：展示审计结果

## 3. 功能模块

### 3.1 审计查询
#### 3.1.1 功能目的
让管理员定位后台操作记录。

#### 3.1.2 触发条件
用户进入页面或修改筛选条件。

#### 3.1.3 前置条件
用户具备系统管理员权限。

#### 3.1.4 主流程
1. 页面展示筛选表单。
2. 用户按请求方法、是否成功、路径、操作人和时间范围筛选。
3. 页面刷新审计列表。

#### 3.1.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.1.6 状态与反馈
- 空态：暂无审计日志
- 失败：提示审计日志加载失败

#### 3.1.7 交互与展示
- 列表需展示方法、路径、状态码、时长、操作人和时间

#### 3.1.8 异常与边界
- 无权限用户只显示权限提示，不显示审计数据

## 4. 页面级共享规则
- 本页不与其他页面共享筛选条件
