# 看板配置

## 1. 需求概要

### 模块定位
看板配置页用于维护可复用的看板定义，是总览页预设筛选和展示规则的治理入口。

### 目标
- 让系统管理员维护看板定义
- 为总览页提供可复用预设
- 把“已落地的 JSON 定义管理”从旧文档的未来规划中独立出来

### 页面范围
- 看板列表与状态筛选
- 新建/编辑看板定义
- 定义校验与删除

### 模块依赖
- **全局规则**：`/prd/需求总览.md` §3 角色与权限、§5.1 共享实体
- **关联模块**：`/prd/modules/dashboard-overview.md` §3.1

### Demo 实现状态

| 功能点 | Demo 状态 | 数据来源 | 说明 |
|--------|-----------|----------|------|
| 看板列表 | ✅ 已实现 | `frontend-v2` + `backend-v2` | 支持按状态过滤 |
| 看板定义编辑 | ✅ 已实现 | 表单 + 后端定义 | 支持新建与编辑 |
| 定义校验 | ✅ 已实现 | 后端校验 | 保存前可先校验 |
| 拖拽式编排 | ❌ 未实现 | — | 当前只有 JSON 定义管理 |

### 一期不做
- 当前不做拖拽式可视化编排
- 当前不做普通用户自定义私有看板

## 2. 页面功能总览
- 看板筛选：按启用状态过滤看板定义
- 看板列表：展示可复用定义并切换当前编辑对象
- 定义编辑：新增、编辑、重置当前表单
- 定义校验：在保存前检查定义合法性

## 3. 功能模块

### 3.1 看板选择与筛选
#### 3.1.1 功能目的
帮助管理员快速定位当前要维护的看板定义。

#### 3.1.2 触发条件
用户进入看板配置页或切换筛选条件。

#### 3.1.3 前置条件
用户具备系统管理员权限。

#### 3.1.4 主流程
1. 页面加载看板列表。
2. 用户按状态筛选可见看板。
3. 用户选择某个看板后，右侧表单加载其当前定义。

#### 3.1.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.1.6 状态与反馈
- 默认态：展示列表与当前选中项
- 空态：没有符合条件的看板

#### 3.1.7 交互与展示
- 左侧为选择栏，右侧为编辑区
- “新建看板”应显式清空当前编辑上下文

#### 3.1.8 异常与边界
- 没有选中项时右侧展示“新建”态
- 筛选结果为空时不显示旧的选中态误导

### 3.2 看板定义编辑与校验
#### 3.2.1 功能目的
维护可被总览页复用的看板定义。

#### 3.2.2 触发条件
用户点击新建、编辑、校验或保存。

#### 3.2.3 前置条件
用户具备系统管理员权限。

#### 3.2.4 主流程
1. 用户填写名称、说明、定义内容和状态。
2. 用户可先触发定义校验。
3. 校验通过后，用户保存看板定义。
4. 保存成功后，列表刷新并保持最新状态。

#### 3.2.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.2.6 状态与反馈
- 加载中：保存和校验动作进入不可重复提交状态
- 成功：提示操作成功
- 失败：展示校验或保存失败原因

#### 3.2.7 交互与展示
- 编辑区区分“新建”和“编辑”两种标题
- 校验和保存动作独立，不强制一次完成

#### 3.2.8 异常与边界
- 校验失败时禁止把无效定义当作成功保存
- 删除动作只对已有对象开放

## 4. 页面级共享规则
- 当前选中的看板定义由本页内部状态维护，不与总览页共享选中对象
