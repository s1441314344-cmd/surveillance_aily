# 告警中心

## 1. 需求概要

### 模块定位
告警中心是摄像头监测与治理能力的运营入口，用于统一查看告警事件，并维护 Webhook 和飞书通知路由。

### 目标
- 集中查看告警事件
- 管理外部通知通道
- 让告警能力从实现细节升级为正式页面模块

### 页面范围
- 告警事件
- Webhook 管理
- 通知路由管理

### 模块依赖
- **全局规则**：`docs/PRD/需求总览.md` §3 角色与权限、§4.3 扩展子域定级
- **关联模块**：`docs/PRD/modules/cameras-monitoring.md` §3.1

### Demo 实现状态

| 功能点 | Demo 状态 | 数据来源 | 说明 |
|--------|-----------|----------|------|
| 告警事件列表 | ✅ 已实现 | 真实后端 | 支持筛选和选中 |
| Webhook 管理 | ✅ 已实现 | 真实后端 | 支持新增和编辑 |
| 飞书通知路由 | ✅ 已实现 | 真实后端 | 支持策略级通知路由维护 |

### 一期不做
- 当前不做复杂告警编排
- 当前不做跨团队通知模板中心

## 2. 页面功能总览
- 告警事件：查看当前和历史告警
- Webhook 管理：维护外部通知目标
- 通知路由：维护按策略分发的通知规则

## 3. 功能模块

### 3.1 告警事件工作台
#### 3.1.1 功能目的
让用户集中查看当前告警状态和处理进度。

#### 3.1.2 触发条件
用户进入告警中心或修改筛选条件。

#### 3.1.3 前置条件
用户具备系统管理员或分析查看者权限。

#### 3.1.4 主流程
1. 页面读取告警事件列表。
2. 用户按状态、严重级别和关键字筛选事件。
3. 用户选择当前告警查看详情上下文。

#### 3.1.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.1.6 状态与反馈
- 默认态：展示事件列表
- 空态：当前没有符合条件的告警
- 失败：提示告警列表加载失败

#### 3.1.7 交互与展示
- 页面顶部显示告警摘要徽标
- 事件列表应突出状态和严重级别

#### 3.1.8 异常与边界
- 筛选重置应恢复默认条件

### 3.2 Webhook 管理
#### 3.2.1 功能目的
维护可用于外部通知分发的 Webhook 目标。

#### 3.2.2 触发条件
用户新建、编辑或选择 Webhook。

#### 3.2.3 前置条件
用户具备系统管理员权限。

#### 3.2.4 主流程
1. 页面读取已有 Webhook 列表。
2. 用户新建或编辑 Webhook。
3. 保存成功后刷新列表。

#### 3.2.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.2.6 状态与反馈
- 成功：提示保存成功
- 失败：提示保存失败

#### 3.2.7 交互与展示
- Webhook 工作台和事件工作台使用同页 tab 切换

#### 3.2.8 异常与边界
- 保存中禁止重复提交

### 3.3 通知路由管理
#### 3.3.1 功能目的
按策略和通知对象维护告警分发规则。

#### 3.3.2 触发条件
用户进入“通知路由”标签或编辑路由。

#### 3.3.3 前置条件
用户具备系统管理员权限。

#### 3.3.4 主流程
1. 页面读取通知路由列表和可选策略。
2. 用户创建或编辑路由。
3. 保存成功后刷新路由列表。

#### 3.3.5 流程图
本功能逻辑线性、无复杂分支，省略流程图。

#### 3.3.6 状态与反馈
- 成功：提示保存成功
- 失败：提示保存失败

#### 3.3.7 交互与展示
- 路由表单需能清晰选择策略和通知目标

#### 3.3.8 异常与边界
- 缺少策略选项时不允许提交无效路由

## 4. 页面级共享规则
- 三个 tab 共享同一页面头部和页面级摘要
- 事件筛选只作用于告警事件 tab，不影响 Webhook 和路由管理

## 5. 数据模型
### 5.1 依赖共享实体
- 告警事件
- Webhook
- 通知路由

### 5.2 页面私有对象
| 字段 | 含义 |
|------|------|
| `selectedAlertId` | 当前选中告警 |
| `selectedWebhookId` | 当前选中 Webhook |
| `selectedNotificationRouteId` | 当前选中通知路由 |
