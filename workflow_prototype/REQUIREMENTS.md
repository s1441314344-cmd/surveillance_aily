# MES工作流编排系统 - 需求规格说明书

## 1. 项目概述

### 1.1 背景
- **业务领域**：饮料生产制造业MES系统
- **当前问题**：使用Camunda BPMN，界面不美观、操作不便、流程编排麻烦
- **目标**：重新设计工作流编排交互，参考飞书多维表工作流风格

### 1.2 用户角色
- 主要用户：工艺工程师、生产管理人员
- 技术水平：熟悉生产流程，但非技术专业

## 2. 功能需求

### 2.1 核心功能清单

#### FR-001: 节点展示
**优先级**: Must Have
- 节点以卡片形式垂直排列展示
- 节点类型：开始、任务、并行网关、结束
- 每个节点显示名称、岗位、参数信息

#### FR-002: 节点添加
**优先级**: Must Have
- 通过点击节点间的"+"按钮添加新节点
- 弹出节点选择器，选择节点类型
- 支持添加：任务节点、并行分支

#### FR-003: 节点编辑
**优先级**: Must Have
- 点击节点上的编辑按钮打开编辑弹窗
- 可编辑：节点名称、岗位、参数列表
- 参数包含：参数名、单位、参数值

#### FR-004: 节点删除
**优先级**: Must Have
- 点击节点上的删除按钮删除节点
- 删除后自动连接上下游节点
- 开始和结束节点不可删除

#### FR-005: 节点拖拽
**优先级**: Should Have
- 可拖拽节点调整位置
- 拖拽后连线自动更新

#### FR-006: 并行流程
**优先级**: Must Have
- 支持并行分支结构
- 并行开始节点（Fork）：一个输入，多个输出
- 并行结束节点（Join）：多个输入，一个输出
- 并行分支清晰展示

#### FR-007: 连线展示
**优先级**: Must Have
- 节点间使用曲线连接
- 连线带箭头指示方向
- 连线清晰不交叉

### 3.3 连线设计
- 类型：贝塞尔曲线
- 颜色：#DEE0E3（默认），#3370FF（选中）
- 带箭头标记
- 连线中间显示"+"按钮（hover时显示）

### 3.4 颜色规范
- 主色：#3370FF（蓝色）
- 成功：#34D399（绿色）- 开始节点

- 边框：#DEE0E3
- 文字：#1F2329（主文字），#8F959E（次要文字）

## 4. 交互规范

### 4.1 添加节点流程
1. 鼠标悬停在连线上，显示"+"按钮
2. 点击"+"按钮，弹出节点选择器
3. 选择节点类型（任务/并行分支）
4. 新节点插入到当前位置

### 4.2 编辑节点流程
1. 点击节点上的"编辑"按钮
2. 弹出编辑弹窗
3. 修改节点信息
4. 点击保存，节点卡片实时更新

### 4.3 删除节点流程
1. 点击节点上的"删除"按钮
2. 确认删除
3. 节点删除，上下游自动连接

### 4.4 拖拽节点流程
1. 按住节点拖动
2. 节点跟随鼠标移动
3. 释放后位置固定
4. 连线自动更新

## 5. 节点类型定义

### 5.1 开始节点
- 图标：绿色圆点
- 名称：开始
- 无输入端口，有输出端口

### 5.2 任务节点
- 图标：根据任务类型变化
- 名称：自定义（加水、搅拌等）
- 有输入端口和输出端口
- 显示岗位和参数

### 5.3 并行开始节点（Fork）
- 图标：菱形，内部"+"
- 名称：并行开始
- 一个输入，多个输出

### 5.4 并行结束节点（Join）
- 图标：菱形，内部"−"
- 名称：并行结束
- 多个输入，一个输出

### 5.5 结束节点
- 图标：红色双圆环
- 名称：结束
- 有输入端口，无输出端口

## 6. 数据结构

### 6.1 节点数据结构
```javascript
{
  id: string,           // 唯一标识
  type: string,         // 节点类型：start|task|parallel_start|parallel_end|end
  name: string,         // 节点名称
  x: number,            // X坐标
  y: number,            // Y坐标
  data: {
    position: string,   // 岗位
    params: [           // 参数列表
      {
        name: string,   // 参数名
        unit: string,   // 单位
        value: string   // 值
      }
    ]
  }
}
```

### 6.2 连线数据结构
```javascript
{
  id: string,       // 唯一标识
  source: string,   // 源节点ID
  target: string    // 目标节点ID
}
```

## 7. 验收标准

### AC-001: 节点展示
- WHEN 页面加载时，THEN 显示完整的工作流图
- WHEN 节点有参数时，THEN 参数直接显示在卡片上

### AC-002: 节点添加
- WHEN 鼠标悬停在连线上，THEN 显示"+"按钮
- WHEN 点击"+"按钮，THEN 弹出节点选择器
- WHEN 选择节点类型后，THEN 新节点插入到正确位置

### AC-003: 节点编辑
- WHEN 点击编辑按钮，THEN 弹出编辑弹窗
- WHEN 修改信息后点击保存，THEN 节点卡片实时更新

### AC-004: 节点删除
- WHEN 点击删除按钮并确认，THEN 节点被删除
- WHEN 节点被删除后，THEN 上下游节点自动连接

### AC-005: 并行流程
- WHEN 添加并行分支时，THEN 正确创建Fork和Join节点
- WHEN 并行分支展示时，THEN 结构清晰可辨
