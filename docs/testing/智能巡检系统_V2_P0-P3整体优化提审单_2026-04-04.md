## 2. 变更摘要
- 本次变更做了什么：
  - 统一 `README_V2.md`、需求、计划、Backlog、验收清单之间的 V2 基线口径，补齐 Owner / 生效日期 / 冻结条件。
  - 新增需求到验证追踪矩阵，收紧 Backlog 关闭标准与提审/缺陷治理要求。
  - 为 `preflight`、`uat`、`verify`、`release-gate`、`security`、`reconcile` 补齐 `run_id`、`git_sha`、`branch`、`started_at`、`finished_at`、`environment`、参数摘要等元数据。
  - 收紧 release gate 与 checklist：禁止隐式“最新文件”回溯，要求显式 `--uat-summary` / `--release-drill-report`，绕过项必须附 `--override-reason`。
  - 后端引入内部任务派发端口，解除 `job_service` / `scheduler_service` 对 worker task 的直接依赖，并将 `alert_service` 拆为 facade + helper 模块。
  - 前端统一 Router / Layout 权限来源，并将 `configCenter.ts` 拆为领域模块后保留兼容 barrel。
  - 补齐前后端回归测试与前端测试环境 `localStorage` mock，使当前分支在本机环境可稳定执行全量验证。
- 未做什么：
  - 未修改对外 HTTP 路由和数据协议。
  - 未执行数据库 schema 迁移。
  - 未新增业务功能，仅做治理、门禁、架构边界和验证闭环优化。
- 是否涉及对外 HTTP API 变化：`否`

## 5. 风险与回滚
- 已知风险：
  - `alert_service` 已拆为 facade + helper，若后续新增 monkeypatch 点位但未走 facade 注入，可能引入测试/运行时漂移。
  - 前端 `configCenter` 现为 barrel 兼容层；仓库外若存在直接依赖全部导出的消费者，仍需在集成环境做一次真实联调。
  - 当前 Node 环境仍会打印 `--localstorage-file` warning，但已通过测试 mock 消除功能性影响。
- 风险等级：`P2`
- 回滚触发条件：
  - 任务派发出现 queued 但未执行。
  - signal monitor 调度出现 silent failure。
  - 告警 webhook / 通知搜索 / 重试链路异常。
  - 前端菜单可见性与页面可访问性不一致。
- 回滚步骤：
  - 回退以下变更组：
    - 后端：`backend-v2/app/services/job_service.py`、`backend-v2/app/services/scheduler_service.py`、`backend-v2/app/services/alert_service.py` 及新增 helper/dispatcher 文件
    - 前端：`frontend-v2/src/shared/auth/permissions.ts`、`frontend-v2/src/app/AppRouter.tsx`、`frontend-v2/src/layouts/AppLayout.tsx`、`frontend-v2/src/shared/api/configCenter.ts` 及 `config-center/`
    - 脚本：`scripts/v2/release-checklist.sh`、`scripts/v2/release-gate.sh`、`scripts/v2/preflight.sh`、`scripts/v2/verify.sh`、`scripts/v2/uat.sh`
  - 回退后重新执行：
    - `PYTEST_DISABLE_PLUGIN_AUTOLOAD=1 pytest backend-v2/tests -q`
    - `npm test`
    - `npm run build`
