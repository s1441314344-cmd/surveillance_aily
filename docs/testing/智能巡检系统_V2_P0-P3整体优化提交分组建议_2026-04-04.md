# 智能巡检系统 V2 P0-P3 整体优化提交分组建议

## 1. 目标

本建议用于把当前工作区中的治理、脚本、架构和测试改动拆成可评审的提交序列，避免一次性大提交导致评审失焦。

默认原则：

- 先文档与基线，再门禁与脚本，再架构边界，再测试与评审材料
- 每个提交都应当独立可解释、可验证、可回滚
- 不将历史文档重组与本轮优化逻辑混在同一提交中

## 2. 建议提交序列

建议标题：

`docs(v2): unify baseline, traceability and governance docs`

建议范围：

- `README_V2.md`
- `docs/INDEX.md`
- `docs/product/`
- `docs/plan/`
- `docs/testing/智能巡检系统_V2_全量验收清单.md`
- `docs/testing/智能巡检系统_V2_提审模板.md`

提交说明：

## 3. 不建议混入本轮提交的内容

- `README.md` 的非 V2 主线说明调整
- 历史 `docs/*.md` 删除与目录重组中不属于本轮治理逻辑的部分
- `backend-v2/README.md`、`frontend-v2/README.md` 的独立整理

这些内容建议单独作为“仓库文档重组”提交，不要与 P0-P3 优化混提。
