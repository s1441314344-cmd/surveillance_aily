# 智能巡检系统 V2 P0-P3 整体优化实际提交命令建议

## 1. 使用原则

- 以下命令用于把当前工作区拆成可评审提交。
- 默认不把仓库历史文档重组、旧 README 删除、旧 `docs/*.md` 删除混入本轮治理提交。

## 3. 不建议混入本轮治理提交的文件

建议单独整理或暂不纳入：

```text
README.md
backend-v2/README.md
frontend-v2/README.md
docs/旧版根目录文档删除记录
```

原因：

- 这些内容属于仓库结构整理或历史材料迁移，不是本轮 P0-P3 治理收敛的核心变更。
- 若与本轮治理提交混合，会显著增加评审噪音。

## 4. 若要一次性快速执行

如果你接受上述拆分顺序，建议按以下节奏执行：

1. `Commit 1`
2. `Commit 2`
3. `Commit 3`
4. `Commit 4`
5. `Commit 5`

每一步都先 `git add`，再运行对应验证，最后再 `git commit`。
