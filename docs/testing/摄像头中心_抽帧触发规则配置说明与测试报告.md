# 摄像头中心：抽帧触发规则配置说明与测试报告

## 1. 功能概述

本次在“摄像头中心”新增了**抽帧触发规则模块**，用于在工厂场景下按可配置条件判断是否触发抓拍，支持以下事件类型：

- `person`：人员出现
- `fire`：疑似着火
- `leak`：疑似漏水
- `custom`：自定义工业事件键

规则支持配置、编辑、删除、启用/禁用、调试（dry-run）和非 dry-run 触发（可选命中即拍照）。

---

## 2. 配置规则定义

### 2.1 规则字段

每条规则包含如下字段：

- `name`：规则名称
- `event_type`：事件类型（`person | fire | leak | custom`）
- `event_key`：信号键（自定义事件必须填写）
- `enabled`：是否启用
- `min_confidence`：置信度阈值（`0 ~ 1`）
- `min_consecutive_frames`：连续命中帧数阈值（`>= 1`）
- `cooldown_seconds`：冷却时间（秒，`>= 0`）
- `description`：规则说明
- `last_triggered_at`：最近触发时间（系统维护）

### 2.2 命中判定逻辑

规则命中需满足以下条件：

1. 规则已启用
2. 输入信号 `signals[event_key] >= min_confidence`
3. 输入连续帧 `consecutive_hits[event_key] >= min_consecutive_frames`
4. 冷却时间满足（距离 `last_triggered_at` 超过 `cooldown_seconds`）

若 `dry_run=false`，命中后会写入 `last_triggered_at`。

若 `capture_on_match=true` 且 `dry_run=false`，首个命中规则会触发一次抓拍，媒体 `source_kind=trigger_rule`。

## 3. 后端接口说明

新增接口均挂载在 `/api/cameras/{camera_id}` 下：

- `GET /trigger-rules`：查询规则列表
- `POST /trigger-rules`：创建规则（管理员）
- `PATCH /trigger-rules/{rule_id}`：更新规则（管理员）
- `DELETE /trigger-rules/{rule_id}`：删除规则（管理员）
- `POST /trigger-rules/debug`：规则调试（管理员/任务操作员）

调试请求体示例：

```json
{
  "signals": { "person": 0.86, "fire": 0.05, "leak": 0.02 },
  "consecutive_hits": { "person": 2, "fire": 1, "leak": 1 },
  "dry_run": true,
  "capture_on_match": false,
  "source_kind": "trigger_rule"
}
```

## 4. 前端操作指南

入口：`摄像头中心 -> 抽帧触发规则`

### 4.1 新增规则

1. 选择目标摄像头
2. 点击“新增规则”
3. 填写规则名称、事件类型、阈值、连续帧、冷却时间
4. 保存后在规则列表可见

### 4.2 调试规则

1. 在“信号输入(JSON)”填写当前信号值
2. 在“连续帧输入(JSON)”填写连续命中帧数
3. 勾选“仅调试不落库”可进行仿真
4. 需要验证真实触发时，关闭 dry-run，并可选择“命中即拍照”
5. 点击“调试规则”查看命中结果和原因

---

## 5. 稳定性与兼容性说明

- 不改变原有任务链路（上传任务、摄像头单次任务、定时任务）
- 规则调试为增量能力，不影响正常状态巡检与媒体管理
- 继续兼容现有 RTSP 抓帧与 mock 摄像头测试模式
