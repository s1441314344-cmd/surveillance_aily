## 一、项目背景与目标

### 1.1 项目背景

当前基于飞书Aily平台的智能巡检系统存在以下问题：
- 依赖Aily平台，流程复杂
- 调试困难，排查问题耗时
- Aily返回结果不直观

### 1.2 项目目标

设计一套**离线智能巡检工具**，不依赖Aily平台，实现：
- 摄像头读取（支持RTSP视频流和本地图片）
- 图像识别（基于YOLOv5目标检测）
- 提示词维护（本地规则配置管理）
- 结果记录（本地数据库存储）
- 友好的Web界面操作

## 二、功能需求清单

### 2.1 核心功能模块

| 功能模块 | 功能编号 | 功能名称 | 功能描述 | 优先级 |
|---------|---------|---------|---------|-------|
| 视频流处理 | F01 | RTSP连接 | 连接RTSP网络摄像头获取视频流 | Must |
| F02 | 图像截取 | 从视频流中定时截取图像帧 | Must |
| F03 | 本地图片 | 支持上传本地图片进行检测 | Must |
| 图像识别 | F04 | YOLOv5检测 | 使用本地YOLOv5模型进行目标检测 | Must |
| F05 | 规则匹配 | 根据配置的规则过滤检测结果 | Must |
| F06 | 结果展示 | 在前端展示检测结果和标注图像 | Must |
| 规则管理 | F07 | 规则增删改查 | 本地CRUD规则配置 | Must |
| F08 | 规则导入导出 | 支持规则的导入导出JSON格式 | Should |

| F13 | 检测参数 | 配置检测频率、阈值等参数 | Must |

### 2.2 功能详细说明

#### F01 - RTSP连接
- 支持rtsp://用户名:密码@IP地址:端口/路径格式
- 支持认证信息配置
- 支持连接超时设置（默认10秒）
- 支持自动重连机制

#### F02 - 图像截取
- 定时截取模式：按配置的频率（如30秒/次）截取
- 手动截取模式：用户手动触发截取
- 截取图片本地临时存储

#### F03 - 本地图片上传
- 支持拖拽上传
- 支持点击选择文件
- 支持格式：JPG、PNG、BMP
- 图片大小限制：20MB

#### F04 - YOLOv5检测
- 本地加载YOLOv5s模型（约14MB）
- 支持80种COCO目标类别
- 支持自定义类别过滤
- 检测阈值可配置（默认0.25）

#### F05 - 规则匹配
- 规则定义：包含点位名称、检测类别、是否启用
- 规则与点位的关联关系
- 检测结果按规则过滤

#### F06 - 结果展示
- 前端实时显示检测结果
- 在图片上标注检测框和类别
- 显示检测数量和置信度

#### F07 - 规则增删改查
- 规则列表页面
- 规则编辑表单（名称、类别、描述）
- 规则启用/禁用开关
- 规则删除确认

#### F09 - 检测记录存储
- SQLite数据库存储
- 记录字段：时间、点位、规则、检测数量、原始图片路径、结果图片路径

#### F10 - 记录查询
- 时间范围筛选
- 点位筛选
- 规则筛选
- 分页展示（默认20条/页）

## 三、数据模型设计

### 3.1 数据库表结构

```sql
-- 点位配置表
CREATE TABLE points (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code VARCHAR(50) UNIQUE NOT NULL,
    name VARCHAR(100) NOT NULL,
    rtsp_url TEXT,
    location VARCHAR(200),
    status VARCHAR(20) DEFAULT 'active',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 规则配置表
CREATE TABLE rules (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code VARCHAR(50) UNIQUE NOT NULL,
    name VARCHAR(100) NOT NULL,
    detect_classes TEXT,  -- JSON数组，如 [0, 2]
    description TEXT,
    threshold FLOAT DEFAULT 0.25,
    status VARCHAR(20) DEFAULT 'active',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 点位-规则关联表
CREATE TABLE point_rule_mapping (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    point_id INTEGER NOT NULL,
    rule_id INTEGER NOT NULL,
    FOREIGN KEY (point_id) REFERENCES points(id),
    FOREIGN KEY (rule_id) REFERENCES rules(id),
    UNIQUE(point_id, rule_id)
);

-- 检测记录表
CREATE TABLE detection_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    point_id INTEGER NOT NULL,
    rule_id INTEGER NOT NULL,
    image_path VARCHAR(500),
    result_image_path VARCHAR(500),
    detection_count INTEGER,
    detection_classes TEXT,  -- JSON数组
    confidence_avg FLOAT,
    detect_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (point_id) REFERENCES points(id),
    FOREIGN KEY (rule_id) REFERENCES rules(id)
);

-- 系统配置表
CREATE TABLE system_config (
    key VARCHAR(100) PRIMARY KEY,
    value TEXT,
    description VARCHAR(200),
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 3.2 主要数据模型类

```
Point (点位)
├── code: 点位编码
├── name: 点位名称
├── rtsp_url: RTSP地址
├── location: 位置描述
├── status: 状态(active/inactive)
└── rules: 关联的规则列表

Rule (规则)
├── code: 规则编码
├── name: 规则名称
├── detect_classes: 检测类别列表[0,2]
├── description: 规则描述
├── threshold: 检测阈值
└── status: 状态

DetectionRecord (检测记录)
├── point_id: 点位ID
├── rule_id: 规则ID
├── image_path: 原始图片路径
├── result_image_path: 结果图片路径
├── detection_count: 检测数量
├── detection_classes: 检测到的类别
├── confidence_avg: 平均置信度
└── detect_time: 检测时间
```

## 六、非功能需求

### 6.1 性能需求
- 图片检测响应时间：< 2秒（YOLOv5s模型）
- RTSP连接建立时间：< 5秒
- 界面操作响应时间：< 500ms
- 同时支持视频流数量：1-4路

### 6.2 可靠性需求
- RTSP断线自动重连：最多重试3次，间隔30秒
- 异常数据不导致程序崩溃
- 数据库操作失败有回滚机制

### 6.3 可维护性需求
- 配置文件与代码分离
- 日志分级输出（DEBUG/INFO/WARNING/ERROR）
- 模块化设计，便于扩展

### 6.4 兼容性需求
- 操作系统：Windows 10+、macOS 10.14+、Ubuntu 18.04+
- 浏览器：Chrome 80+、Firefox 75+、Safari 13+
- Python版本：3.8+
