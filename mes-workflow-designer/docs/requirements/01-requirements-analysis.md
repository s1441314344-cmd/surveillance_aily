# MES 工艺流程编辑器 - 需求分析文档

**版本**: v1.0  
**日期**: 2026-03-16  
**状态**: 需求确认

---

## 1. 项目概述

### 1.1 项目背景
开发一个专为制造业MES系统设计的两层架构工艺流程编辑器，支持研发工艺设计与生产工艺实例化的完整工作流。

### 1.2 核心目标
- 实现研发工艺层：多工艺一张图，支持分组和跨组引用
- 实现生产工艺层：引用研发模板，拆分生成独立BPMN图
- 建立两层之间的关联与转换机制

## 2. 功能需求

### 2.1 研发工艺层（RecipeTemplate Layer）

#### FR-001: 可视化图编辑
**优先级**: Must Have

**描述**: 提供直观的图形化界面，支持直接拖拽编辑工艺流程图

**详细需求**:
1. 画布支持无限滚动和缩放
2. 节点支持拖拽创建和移动
3. 连线支持自动路由和手动调整
4. 支持撤销/重做操作

**验收标准**:
- WHEN 用户拖拽节点到画布, the system SHALL 在指定位置创建节点
- WHEN 用户连接两个节点, the system SHALL 创建带箭头的连线
- WHEN 用户拖拽节点, the system SHALL 实时更新连线位置

#### FR-002: 工艺分组管理
**优先级**: Must Have

**描述**: 支持将工艺节点按类型分组，不同组使用不同视觉主题

**详细需求**:
1. 支持创建多个工艺组（如溶解组、调配组）
2. 每组有独立的视觉主题（颜色、边框样式）
3. 组内节点自动归属，支持跨组移动
4. 组可以折叠/展开

**验收标准**:
- WHEN 用户创建工艺组, the system SHALL 生成带主题色的分组区域
- WHEN 用户将节点移入分组, the system SHALL 更新节点所属组
- WHEN 用户点击折叠按钮, the system SHALL 收起该组所有节点

#### FR-003: 跨组引用关系
**优先级**: Must Have

**描述**: 支持不同工艺组之间的节点引用，建立虚线连接关系

**详细需求**:
1. 支持创建跨组引用连线（虚线表示）
2. 被引用节点变更时，所有引用节点同步更新
3. 引用关系可视化展示
4. 支持查看引用来源和去向

**验收标准**:
- WHEN 用户创建跨组引用, the system SHALL 用虚线连接两个节点
- WHEN 被引用节点参数变更, the system SHALL 同步更新所有引用节点
- WHEN 用户悬停引用线, the system SHALL 高亮显示引用关系

#### FR-004: 节点参数配置
**优先级**: Must Have

**描述**: 节点支持多层级参数配置（工艺-工序-参数）

**详细需求**:
1. 节点三级结构：工艺类型 → 工序名称 → 具体参数
2. 参数类型支持：数值、范围、选项、文本
3. 参数单位自动转换和校验
4. 支持参数模板快速填充

**验收标准**:
- WHEN 用户编辑节点, the system SHALL 打开参数配置面板
- WHEN 用户输入参数值, the system SHALL 实时校验有效性
- WHEN 用户选择工艺模板, the system SHALL 自动填充默认参数

#### FR-005: 研发工艺保存与版本
**优先级**: Should Have

**描述**: 支持研发工艺的保存、版本管理和历史回溯

**详细需求**:
1. 自动保存草稿
2. 手动发布版本
3. 版本历史查看和回滚
4. 版本对比功能

### 2.2 生产工艺层（RecipeInstance Layer）

#### FR-006: 引用研发模板
**优先级**: Must Have

**描述**: 生产工艺可以引用研发工艺模板作为基础

**详细需求**:
1. 从研发模板列表选择引用
2. 继承研发模板的所有节点和参数
3. 显示模板版本信息
4. 支持切换模板版本

**验收标准**:
- WHEN 用户选择研发模板, the system SHALL 加载模板数据
- WHEN 模板有更新, the system SHALL 提示用户同步
- WHEN 用户确认同步, the system SHALL 更新实例数据

#### FR-007: 工艺拆分生成
**优先级**: Must Have

**描述**: 根据研发工艺的分组，拆分为独立的生产工艺流程

**详细需求**:
1. 按工艺组自动拆分
2. 每个组生成独立的流程图
3. 保留组内连线和引用关系
4. 支持调整拆分规则

**验收标准**:
- WHEN 用户点击生成功能, the system SHALL 按组拆分工艺
- WHEN 拆分完成, the system SHALL 为每个组创建独立画布
- WHEN 用户查看拆分结果, the system SHALL 显示组内完整流程

#### FR-008: 参数实例化
**优先级**: Must Have

**描述**: 生产层继承研发参数，支持实例级覆盖

**详细需求**:
1. 自动带出研发层定义的参数
2. 支持修改参数值（标记为覆盖）
3. 显示参数来源（继承/覆盖）
4. 支持重置为模板值

**验收标准**:
- WHEN 生成生产工艺, the system SHALL 自动填充研发参数
- WHEN 用户修改参数, the system SHALL 标记为覆盖状态
- WHEN 用户点击重置, the system SHALL 恢复模板值

#### FR-009: BPMN 2.0 导出
**优先级**: Must Have

**描述**: 将生产工艺导出为标准BPMN 2.0格式

**详细需求**:
1. 符合BPMN 2.0 XML规范
2. 支持任务、网关、事件等标准元素
3. 保留工艺参数作为扩展属性
4. 支持导入验证

**验收标准**:
- WHEN 用户导出BPMN, the system SHALL 生成标准XML文件
- WHEN BPMN文件导入其他系统, the system SHALL 正确解析
- WHEN 导出包含参数, the system SHALL 使用BPMN扩展机制

- 自动保存间隔 30秒

- 支持Chrome、Firefox、Safari最新版本
- 响应式布局，支持1920x1080及以上分辨率

- 用户认证和授权
- 操作日志记录

## 4. 用户故事

### US-001: 研发工程师设计工艺
> **As a** 研发工程师  
> **I want** 在画布上拖拽创建工艺节点并分组  
> **So that** 我可以直观地设计完整的研发工艺

**验收标准**:
- GIVEN 我在研发工艺编辑页面
- WHEN 我从工具栏拖拽节点到画布
- THEN 节点出现在指定位置并属于默认组

### US-002: 建立跨组引用
> **As a** 研发工程师  
> **I want** 在不同工艺组之间建立引用关系  
> **So that** 我可以复用溶解工艺的节点到调配工艺

**验收标准**:
- GIVEN 溶解组和调配组都有节点
- WHEN 我创建从调配组节点到溶解组节点的引用
- THEN 显示虚线连接，且溶解节点变更时调配节点同步更新

### US-003: 生产工程师生成工艺
> **As a** 生产工程师  
> **I want** 引用研发模板并拆分为生产工艺  
> **So that** 我可以基于研发设计生成可执行的生产流程

**验收标准**:
- GIVEN 已有研发工艺模板
- WHEN 我选择模板并点击生成
- THEN 系统按组拆分，每个组生成独立的BPMN图

### US-004: 调整生产参数
> **As a** 生产工程师  
> **I want** 修改继承的参数值  
> **So that** 我可以根据实际生产情况调整工艺参数

**验收标准**:
- GIVEN 我已生成生产工艺
- WHEN 我修改某个参数值
- THEN 参数标记为覆盖，但可以随时重置回模板值

## 6. 数据模型

### 6.1 核心实体关系

```
RecipeTemplate (研发工艺模板)
├── id: string (唯一标识)
├── name: string (工艺名称)
├── version: number (版本号)
├── groups: ProcessGroup[] (工艺组列表)
├── nodes: ProcessNode[] (节点列表)
├── edges: ProcessEdge[] (连线列表)
├── createdAt: DateTime
└── updatedAt: DateTime

ProcessGroup (工艺组)
├── id: string
├── name: string (组名：溶解、调配等)
├── color: string (主题色)
├── nodes: string[] (包含的节点ID列表)
└── position: { x, y, width, height }

ProcessNode (工艺节点)
├── id: string
├── name: string (节点名称：加水、搅拌等)
├── groupId: string (所属组ID)
├── type: 'task' | 'gateway' | 'event'
├── position: { x, y }
├── parameters: Parameter[] (参数列表)
├── isReference: boolean (是否引用节点)
├── refNodeId: string | null (引用的源节点ID)
└── refGroupId: string | null (引用的源组ID)

Parameter (工艺参数)
├── id: string
├── name: string (参数名：温度、容量等)
├── value: any (参数值)
├── unit: string (单位)
├── type: 'number' | 'range' | 'select' | 'text'
└── options: any[] (选项值，type=select时使用)

ProcessEdge (工艺连线)
├── id: string
├── source: string (源节点ID)
├── target: string (目标节点ID)
├── type: 'intra-group' | 'inter-group' (组内/跨组)
└── style: 'solid' | 'dashed' (实线/虚线)

RecipeInstance (生产工艺实例)
├── id: string
├── templateId: string (引用的模板ID)
├── templateVersion: number (引用的模板版本)
├── name: string (实例名称)
├── processes: ProcessInstance[] (工艺实例列表)
└── createdAt: DateTime

ProcessInstance (工艺实例)
├── id: string
├── groupId: string (对应的组ID)
├── name: string (工艺名称)
├── nodes: NodeInstance[] (节点实例列表)
├── edges: EdgeInstance[] (连线实例列表)
└── bpmnXml: string (BPMN XML内容)

NodeInstance (节点实例)
├── id: string
├── templateNodeId: string (模板节点ID)
├── name: string
├── parameters: ParameterInstance[] (参数实例)
└── position: { x, y }

ParameterInstance (参数实例)
├── id: string
├── templateParamId: string (模板参数ID)
├── name: string
├── inheritedValue: any (继承值)
├── currentValue: any (当前值)
├── isOverridden: boolean (是否被覆盖)
└── unit: string
```

## 7. 技术栈建议

### 7.1 前端技术栈
- **框架**: React 18 + TypeScript
- **状态管理**: Zustand
- **图编辑**: ReactFlow (基于BPMN的可视化)
- **UI组件**: Ant Design 或 Chakra UI
- **BPMN处理**: bpmn-js (用于BPMN 2.0导入导出)

### 7.2 后端技术栈
- **框架**: Node.js + Express 或 NestJS
- **数据库**: PostgreSQL (主数据) + Redis (缓存)
- **ORM**: Prisma
- **API**: RESTful + WebSocket (实时协作)

### 7.3 部署
- **容器化**: Docker + Docker Compose
- **前端部署**: Nginx
- **后端部署**: PM2

## 9. 风险与约束

### 9.1 技术风险
- BPMN 2.0规范复杂，需要充分测试兼容性
- 大量节点时的性能优化

### 9.2 业务风险
- 用户对两层架构的理解成本
- 工艺参数的标准化定义

### 9.3 约束条件
- 必须符合BPMN 2.0标准
- 支持主流浏览器
